KinetTask Privacy Policy

Last updated: June 16, 2026
✓ We do not collect your data

KinetTask is an AI task manager that runs 100% on your device. This policy explains honestly how KinetTask handles your information. The short version: your tasks, notes, and all AI processing happen only on your device. We collect nothing, upload nothing, and track no one.

In short: KinetTask has no account, no cloud backend, no analytics SDK, no advertising, and no tracking. The only network activity is downloading public model files when you explicitly request them — it never sends your content anywhere.

1. Information We Collect

None. KinetTask does not collect any personal data, including:

  • No usage or analytics data
  • No crash reporting (no crash-reporting SDK is integrated)
  • No advertising tracking or user profiling (no IDFA, no device fingerprinting)
  • No registration or login — there is no account system
  • Your tasks, notes, tags, or anything you type are never uploaded

2. Where Your Data Lives

All of your task data is stored in a local database on your device, inside the app's sandbox container:

  • tasks.sqlite3 — task data
  • search.sqlite3 — full-text search index (FTS5)
  • projects.sqlite3 — project data

macOS: ~/Library/Containers/com.kitnet.task.app/Data/Library/Application Support/KinetTask/
These files belong to you. You can back them up at any time with Time Machine or any backup tool.

3. How the AI Works (Fully On-Device)

KinetTask's AI features (natural-language parsing, task breakdown, related-task suggestions, daily insights) run entirely on your device:

  • Apple Foundation Models — the system's built-in on-device model (macOS 26 / iOS 26).
  • Local GGUF models (Llama.cpp) — a model you choose to download, running in-process on your device.

The App Store build offers no remote (cloud) AI. Your task content never leaves the device for AI processing. Even with no model loaded, KinetTask falls back to built-in rules for basic date and priority parsing.

4. Network Usage (Stated Honestly)

KinetTask works offline by default. It makes a network request only when you actively download a model file, fetching public, open-source model files that contain none of your data:

ActivityNetwork?Details
Creating, editing, deleting tasks; AI parsing; searchNoFully offline
Downloading a GGUF modelYesFetches a public model file from HuggingFace (optionally via the hf-mirror.com mirror). Only triggered by you; sends no user data.

Aside from the above, KinetTask makes no outbound network connections. The source code is auditable.

5. Permissions

KinetTask does not request camera, microphone, location, contacts, photos, health, or any other sensitive system permission. If you import a local GGUF model via the file picker, the app reads only that file within the scope you grant.

6. Third-Party Components

KinetTask uses the following open-source components, none of which transmit data:

  • LocalLLMKit (MIT) — local LLM inference abstraction, runs on-device.
  • SQLite.swift (MIT) — local database wrapper.
  • Apple FoundationModels (system framework) — on-device AI.

Downloaded models are open-weight models (such as Qwen2.5 / Qwen3) governed by their respective open-source licenses.

7. Data Security

Because all data stays in the local app sandbox and is protected by the macOS / iOS app sandbox and system encryption, there is no risk of your data being transmitted to us or any third-party server. App Sandbox and Hardened Runtime are enabled.

8. Your Rights (Your Data Belongs to You)

  • Export: Your data is the local database file listed above — copy it to back it up anytime.
  • Delete: Delete tasks inside the app, or simply uninstall the app to erase all data completely.
  • No account to delete: KinetTask has no account, so there is no account-deletion step required.

9. Children

KinetTask is not directed at children under 13 and does not knowingly collect any information from children. The App Store age rating is 4+.

10. iCloud Sync

The current version (1.x) does not offer iCloud sync. Related code exists but is disabled by default. If a future version provides it, sync would occur through your private iCloud database under your Apple ID, encrypted end-to-end by Apple, and only if you explicitly enable it. We will update this policy before that happens. You could turn it off at any time in System Settings.

11. Changes to This Policy

If we make material changes to this policy, we will notify you with the app update or on this page. Continued use constitutes acceptance of the updated policy.
